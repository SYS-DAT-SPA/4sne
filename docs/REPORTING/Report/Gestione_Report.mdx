# Gestione Report

![report_icona.jpg](/UFR/imgs/report_icona.jpg)

 Gestione Reports Il **Report** è la omponente del tool che definisce e rappresenta il tipo di risultato. Con un report è possibile configurare stampe, reportistica, tabelle e grafici dinamici a partire da differenti basi di dati. Ogni report può essere inserito a menu per essere utilizzato dagli utenti oppure essere inserito all'interno di una [**Dashboard**](/UFR/imgs/Dashboard/UFR_Creare_gestire_dashboard). --- Possono essere configurate le seguenti tipologie di report. 

![elenco_reports.jpg](/UFR/imgs/elenco_reports.jpg)

 --- In questa sezione troverai le informazioni per poter [**gestire un Report**](/UFR/imgs/Report/Gestione_Reports)

# Configurazione e manutenzione di un report.  

In questa sezione troverai informazioni relative alla configurazione di un report.  ---   La configurazione di un report prevedete una parte di definizione ed una parte di comportamento. La parte di definizione serve a indicare la tipologia di report, caratteristiche e la fonte dati. Mentre la parte comportamentale intoduce elementi quali: filtri, filtri di default, eventuale ordinamento dell'output.  

![ufr_nuovo_report_02.jpg](/UFR/imgs/ufr_nuovo_report_02.jpg)



Dati generali - Identificazione report  Alla creazione di un report occorre inserire un codice (non modificabile successivamente), ed una descrizione. Questi due elementi identificano il report e lo rendono ricercabile all'interno del sistema.   

![ufr_report_codice.jpg](/UFR/imgs/ufr_report_codice.jpg)

 

Dati Applicativi Ad un report deve essere associato un **ApplicationModule**. Questo dato permette di raggruppare i report per modulo applicativo di Next; Consente inoltre l'estrazione e l'esecuzione individuale del report all'indirizzo reports/application-module-name.    

![ufr_report_application.jpg](/UFR/imgs/ufr_report_application.jpg)

 Opzionalmente, è possibile specificare codice azienda di riferimento.  Inoltre, con il campo dell'immagine sotto è possibile caricare un file per la reportistica creata con *Crystal Report*. 

![ufr_report_cristal.jpg](/UFR/imgs/ufr_report_cristal.jpg)

   

Tipo e Formato report       

![ufr_report_tipologia.jpg](/UFR/imgs/ufr_report_tipologia.jpg)



###   Definizione del tipo report Il tipo report rdefinisce la sorgente dei dati dalla quale dovranno essere estratti i dati da elaborare.    

 1. Provider Servizio: i dati vengono estratti tramite chiamata REST ad un service, estraendo da un dataset di cui si richiede la tipologia, definiendo l'indirizzo del Service, l'URL ed il metodo HTTP.  

 2. Provider Database: i dati vengono estratti da un DB. Vanno perciò definiti i parametri per l'apertura della connessione  

 3. Provider comando SQL: i dati vengono estratti da una query definita dall'utente nel DB  4SNE dell'azienda. Con questa tipologia di report è quindi necessario gestire opportunamente una [**Query SQL**](/UFR/imgs/Report/Gestione_Report_Comandi_SQL).  

 4. Provider DataSet Diretto:     

### Definizione del Formato:   Il formato rappresenta la tipologia di presentazione dell'informazione. Attualmente sono supportati i seguenti formati:     

 1. [**Excel**](/UFR/imgs/Report/Report_Excel): l'informazione viene rappresentata come tabella all'interno di un foglio Microsoft Excel (estensione xlsx).  

 2. [**PDF**](/UFR/imgs/Report/Report_PDF): l'informazione viene rappresentata come documento PDF.   

 3. [**Tabella**](/UFR/imgs/Report/Report_Tabella): l'informazione viene rappresentata come tabella interattiva.    

 4. [**Tabella e collegamento**:](/UFR/imgs/Report/Report_Tabella_Collegata) l'informazione viene rappresentata come tabella interattiva con la possibilità di aprire una sottotabella di **drill-down**.

 1.   [**Tabella Pivot**](/UFR/imgs/Report/Tabella_pivot): l'informazione viene rappresentata in una tabella Pivot già formattata.

 5.   [**Grafico**](/UFR/imgs/Report/Tipologie_Grafici). L'informazione viene esposta sotto forma di grafico.
